API ENDPOINTS
================

POST /api/login
- Login with username
- Returns: {success, username, message}

POST /api/submit-score
- Submit game score
- Params: username, game_name, score
- Returns: {success, score_id, score_data}

GET /api/leaderboard
- Get top scores
- Params: game_name (optional), limit (optional)
- Returns: {success, leaderboard[]}

GET /api/games/<game_name>/scores
- Get all scores for a game
- Returns: {success, game_name, total_scores, scores[]}

GET /api/games/<game_name>/user/<username>/scores
- Get user's scores for a game
- Returns: {success, game_name, username, scores[]}

DATABASE SCHEMA
==================

USERS TABLE:
- id (Primary Key)
- username (Unique, String)
- created_at (Timestamp)
- Relationship: One-to-Many with Scores

SCORES TABLE:
- id (Primary Key)
- user_id (Foreign Key to Users)
- username (String, Indexed)
- game_name (String, Indexed)
- score (Integer)
- timestamp (Datetime, Indexed)
