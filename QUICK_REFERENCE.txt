✅ DEBUGGING CHECKLIST
=====================

Issue: Server won't start
  □ Check Python version (3.7+): python --version
  □ Check Flask installation: pip list | grep Flask
  □ Try different port: Edit app.run(port=5001)
  □ Kill process on port 5000: netstat -ano | findstr 5000

Issue: Database errors
  □ Delete minigames.db and restart
  □ Check database permissions
  □ Verify SQLAlchemy installation

Issue: Static files not loading
  □ Hard refresh browser (Ctrl+F5)
  □ Check file paths in templates ({{ url_for('static', ...) }})
  □ Verify files exist in static/ directories

Issue: Games not working
  □ Open browser console (F12)
  □ Check JavaScript errors
  □ Verify game.html loads correct game script
  □ Check GAME_NAME variable is set correctly

Issue: Scores not saving
  □ Check browser console for API errors
  □ Verify /api/submit-score endpoint works
  □ Check database is created
  □ Verify username is not empty

✅ SUPPORT & HELP
=================

Common Questions:

Q: How do I change the game difficulty?
A: Edit the JavaScript files in static/js/ to adjust parameters

Q: Can I add more games?
A: Yes, follow the same structure as existing games

Q: How do I deploy this to production?
A: Use a WSGI server (Gunicorn) and reverse proxy (Nginx)

Q: How can I add password authentication?
A: Add password field to User model and use bcrypt hashing

Q: How do I backup the database?
A: Copy minigames.db file or export using sqlite3 CLI

Q: Can I run multiple instances?
A: Yes, on different ports with separate databases

Q: How do I clear all scores?
A: Delete minigames.db and restart server
